# Zotero Summary Creator

Generate AI-powered summaries from your Zotero research papers using local LLM (Ollama).

**Zotero Plugin** - Right-click any paper in Zotero to generate summaries with innovation opportunities.

---

## Quick Start

### 1. Install Ollama
```bash
curl -fsSL https://ollama.ai/install.sh | sh
ollama pull llama2
```

### 2. Build & Install Plugin
```bash
cd zotero-summary-creator
npm install
npm run build
```

In Zotero:
1. **Tools ‚Üí Add-ons**
2. **Gear icon ‚Üí Install Add-on From File...**
3. Select `build/zotero-summary-creator.xpi`
4. **Restart Zotero**

### 3. Configure
1. **Tools ‚Üí Summary Creator Preferences**
2. Set Ollama URL: `http://localhost:11434`
3. Set Model: `llama2`
4. Click **Test Connection**

### 4. Use It
- **Right-click any paper ‚Üí "Generate Summary"**
- Wait 10-30 seconds
- Summary appears as a note!

## Installation Guide

### Prerequisites
- **macOS** (Windows/Linux untested but may work)
- **Zotero 6 or 7**
- **Node.js** (to build the plugin)
- **Ollama** (local LLM service)

### Step 1: Install Ollama

**Option A: Direct Download**
- Download from https://ollama.ai

**Option B: Command Line**
```bash
curl -fsSL https://ollama.ai/install.sh | sh
```

**Pull a model:**
```bash
# Start Ollama (may auto-start after installation)
ollama serve

# In another terminal, pull a model:
ollama pull llama2        # Good balance (recommended)
# ollama pull mistral     # Better quality, larger
# ollama pull codellama   # Good for technical papers
```

**Verify it's running:**
```bash
curl http://localhost:11434/api/tags
```

Should return JSON with your installed models.

### Step 2: Build the Plugin

```bash
# Clone or download this repository
cd zotero-summary-creator

# Install dependencies
npm install

# Build the plugin
npm run build
```

This creates `build/zotero-summary-creator.xpi`

### Step 3: Install in Zotero

1. **Open Zotero**
2. Go to **Tools ‚Üí Add-ons** (or ‚åò‚áßA on macOS)
3. Click the **gear icon** (‚öôÔ∏è) in the top-right
4. Select **"Install Add-on From File..."**
5. Navigate to `build/zotero-summary-creator.xpi`
6. Click **"Open"**
7. Click **"Install Now"** when prompted
8. **Restart Zotero** when prompted

### Step 4: Configure Settings

1. Open Zotero
2. Go to **Tools ‚Üí Summary Creator Preferences...**
3. Configure:
   - **Ollama URL**: `http://localhost:11434` (default)
   - **Model Name**: `llama2` (or your preferred model)
   - **Auto-open notes**: ‚úÖ Check to automatically open summaries
4. Click **"Test Connection"** to verify Ollama is accessible
5. Click **OK**

## How to Use

### Generate Summary for One Paper
1. **Right-click** on any paper in your library
2. Select **"Generate Summary"**
3. Wait 10-30 seconds (depending on model and paper length)
4. Summary appears as a note attached to the paper

### Generate Summaries for Multiple Papers (Batch Mode)
1. **Select multiple papers** (hold ‚åò/Ctrl and click)
2. **Right-click** on selection
3. Select **"Generate Summary"**
4. Progress window shows "Generating Summaries (Batch Mode)"
5. All summaries generated in sequence

**Batch Mode Behavior:**
- ‚úÖ **Continues on errors** - If one paper fails, the rest still process
- ‚úÖ **No auto-open** - Notes are created but windows don't pop up (avoids opening dozens of windows)
- ‚úÖ **Shows progress** - Each item shows ‚úì (success) or ‚úó (error) in progress window
- ‚úÖ **Error summary** - Final message shows: "Success: X, Failed: Y"
- ‚ÑπÔ∏è **First error alert** - Shows alert for first error only (not every error)

### View Summaries
- Click the paper in your library
- Look in the **Notes** section in the right panel
- Double-click the note to open in editor

### Single Item vs Batch Mode

| Feature | Single Item | Batch Mode (2+ items) |
|---------|-------------|----------------------|
| **Auto-open note** | ‚úÖ Yes (if enabled in prefs) | ‚ùå No (prevents window spam) |
| **Error behavior** | ‚ùå Stops, shows error | ‚úÖ Continues to next item |
| **Progress window** | Shows single item | Shows all items with ‚úì/‚úó |
| **Error alerts** | Shows immediately | Shows for first error only |
| **Use case** | Quick interactive summary | Process entire collection |

### Using Different Content Sources
The plugin automatically:
1. **Checks for abstract** first (fastest)
2. **Falls back to PDF** if no abstract available
3. **Extracts PDF text** using Zotero's built-in indexing
4. **Triggers indexing** if PDF not yet indexed

## Troubleshooting

### Connection Issues

**Problem: "Connection failed" in preferences**

Solutions:
```bash
# Check if Ollama is running
curl http://localhost:11434/api/tags

# If not, start it:
ollama serve

# Verify the model exists:
ollama list

# Pull model if missing:
ollama pull llama2
```

### Summary Generation Fails

**Problem: No summary generated or error message**

Common causes:
1. **No content available** - Paper needs abstract or PDF attachment
2. **Model not found** - Run `ollama pull llama2`
3. **Ollama crashed** - Restart: `ollama serve`
4. **Out of memory** - Use smaller model: `ollama pull llama2:7b`

**Debug steps:**
1. Enable debug logging: **Help ‚Üí Debug Output Logging ‚Üí View Output**
2. Click **"Enable"** and **"Clear Output"**
3. Try generating summary again
4. Look for error messages in debug output

**Test Ollama manually:**
```bash
curl http://localhost:11434/api/generate -d '{
  "model": "llama2",
  "prompt": "Say hello",
  "stream": false
}'
```

### PDF Content Issues

**Problem: "PDF content empty or null"**

Solutions:
1. Right-click PDF in Zotero ‚Üí **"Reindex Item"**
2. Wait for indexing to complete (check indexing progress in Zotero)
3. Try generating summary again

**Problem: Auto-downloaded PDFs (titled "Full Text PDF") don't work**

The plugin now has advanced handling for auto-downloaded PDFs:
- ‚úÖ Checks if PDF file exists on disk (not just a link)
- ‚úÖ Verifies file size to ensure download completed
- ‚úÖ Clears partial/corrupted indexes
- ‚úÖ Force-triggers indexing with exponential backoff
- ‚úÖ Waits up to 30 seconds for indexing (2s, 4s, 8s, 16s intervals)
- ‚úÖ Detects scanned/image PDFs (< 100 chars extracted)

**Advanced PDF diagnostics:**
1. **Tools ‚Üí Developer ‚Üí Run JavaScript**
2. Paste this one-liner:
```javascript
const item = Zotero.getActiveZoteroPane().getSelectedItems()[0];
const att = await Zotero.Items.getAsync(item.getAttachments()[0]);
const path = att.getFilePath();
const exists = path && Zotero.File.pathToFile(path).exists();
const state = await Zotero.Fulltext.getIndexedState(item.getAttachments()[0]);
return `Path: ${path}\nExists: ${exists}\nIndexed: ${state} (0=no, 1=yes, 2=partial)`;
```

Common issues:
- **"Path: null"** ‚Üí PDF not downloaded yet, only a placeholder
- **"Exists: false"** ‚Üí Download incomplete or failed
- **"Indexed: 0"** ‚Üí Needs indexing (plugin will auto-handle this)
- **"Indexed: 2"** ‚Üí Partial index (plugin clears and re-indexes)

**Force reindex all selected items:**
1. **Tools ‚Üí Developer ‚Üí Run JavaScript**
2. Paste this code:
```javascript
var items = Zotero.getActiveZoteroPane().getSelectedItems();
for (let item of items) {
    let attachments = item.getAttachments();
    await Zotero.Fulltext.indexItems(attachments);
}
return "Reindexed " + items.length + " items";
```

### Slow Performance

Solutions:
- Use a smaller/faster model: `ollama pull llama2:7b`
- Summarize from abstracts only (much faster than PDFs)
- Close other applications to free up memory
- Upgrade to GPU-accelerated hardware

### Plugin Doesn't Appear

Solutions:
1. Restart Zotero after installation
2. Check **Tools ‚Üí Add-ons** - should show "Zotero Summary Creator"
3. If missing, reinstall the .xpi file
4. Check Zotero version (requires 6.0+)
5. Check **Tools ‚Üí Developer ‚Üí Error Console** for errors

### Linked Files Not Working (OpenReview, arXiv, etc.)

**Problem: "No content available" for PDFs that exist on disk**

This happens when PDFs are **linked files** instead of **stored/imported files**. Zotero may not index linked files.

**Quick Fix - Enable Linked File Indexing:**
1. **Zotero ‚Üí Preferences ‚Üí Search**
2. Check **"Index linked files"** or **"Index attachments using..."**
3. Right-click collection ‚Üí **"Reindex Items..."**
4. Wait for indexing to complete
5. Try generating summaries again

**Alternative - Convert to Stored Files:**

Quick test: Right-click one PDF ‚Üí **"Store Copy of File"** ‚Üí Try summary again

**Batch Convert Script:** See [Advanced Troubleshooting](#advanced-troubleshooting) section below for a script to convert all linked PDFs to stored PDFs automatically.

### Empty or Incomplete Summaries

**Problem: Summary only has executive summary, missing other sections**

Solutions:
1. Check debug output for "Warning - No key contributions extracted"
2. Try a larger/better model: `ollama pull mistral`
3. Verify paper has substantial content (not just a short abstract)
4. Check Ollama logs: `ollama serve` (look for errors)

## Advanced Troubleshooting

### Debug Auto-Downloaded PDFs

If auto-downloaded PDFs (titled "Full Text PDF") aren't working, use this diagnostic to understand why.

**Run Full Diagnostic:**
1. **Select a paper** with auto-downloaded PDF in Zotero
2. **Tools ‚Üí Developer ‚Üí Run JavaScript**
3. **Paste and run:**

```javascript
async function diagnosePDF() {
  const item = Zotero.getActiveZoteroPane().getSelectedItems()[0];
  if (!item) return "ERROR: No item selected";

  const results = [];
  results.push("=== PDF DIAGNOSTIC ===");
  results.push(`Item ID: ${item.id}`);
  results.push(`Title: ${item.getField('title')}`);
  results.push("");

  const abstract = item.getField('abstractNote');
  results.push(`Abstract: ${abstract ? abstract.length + ' chars' : 'NONE'}`);
  results.push("");

  const attachments = item.getAttachments();
  results.push(`Total attachments: ${attachments.length}`);
  results.push("");

  for (let i = 0; i < attachments.length; i++) {
    const attID = attachments[i];
    const att = await Zotero.Items.getAsync(attID);

    results.push(`--- Attachment ${i + 1} ---`);
    results.push(`  ID: ${attID}`);
    results.push(`  Title: ${att.getField('title')}`);
    results.push(`  Type: ${att.attachmentContentType}`);
    results.push(`  Link mode: ${att.attachmentLinkMode}`);

    const filePath = att.getFilePath();
    results.push(`  File path: ${filePath || 'NONE'}`);

    if (filePath) {
      const file = Zotero.File.pathToFile(filePath);
      const exists = file && file.exists();
      results.push(`  File exists: ${exists}`);
      if (exists) results.push(`  File size: ${file.fileSize} bytes`);
    }

    if (att.attachmentContentType === 'application/pdf') {
      const indexed = await Zotero.Fulltext.getIndexedState(attID);
      const states = { 0: 'UNINDEXED', 1: 'INDEXED', 2: 'PARTIAL' };
      results.push(`  Index state: ${indexed} (${states[indexed] || 'UNKNOWN'})`);

      const content = await Zotero.Fulltext.getItemContent(attID);
      if (content && content.content) {
        results.push(`  Extracted text: ${content.content.length} chars`);
        results.push(`  First 100 chars: ${content.content.substring(0, 100)}`);
      } else {
        results.push(`  Extracted text: NONE`);
      }
    }
    results.push("");
  }

  return results.join('\n');
}

return await diagnosePDF();
```

**Common Issues:**
- `File path: NONE` ‚Üí PDF not downloaded, only placeholder. Wait for download.
- `Index state: 0 (UNINDEXED)` ‚Üí Right-click PDF ‚Üí Reindex Item, wait 30 seconds
- `Extracted text: 0 chars` ‚Üí Scanned/image PDF with no extractable text
- `File exists: false` ‚Üí Download incomplete or failed

### Convert Linked Files to Stored Files

Run this script to convert ALL linked PDFs in selected items to stored files:

1. **Select all items** with linked PDFs
2. **Tools ‚Üí Developer ‚Üí Run JavaScript**
3. **Paste and run:**

```javascript
async function convertLinkedToStored() {
  const items = Zotero.getActiveZoteroPane().getSelectedItems();
  const results = [];
  let converted = 0, skipped = 0, failed = 0;

  for (const item of items) {
    if (!item.isRegularItem()) continue;
    const attachments = item.getAttachments();

    for (const attID of attachments) {
      const att = await Zotero.Items.getAsync(attID);

      if (att.attachmentContentType === 'application/pdf') {
        const linkMode = att.attachmentLinkMode;
        const title = att.getField('title');

        if (linkMode === 1) {  // Linked file
          try {
            const path = att.getFilePath();
            if (!path) {
              results.push(`‚ùå ${title}: No file path`);
              failed++;
              continue;
            }

            const file = Zotero.File.pathToFile(path);
            if (!file.exists()) {
              results.push(`‚ùå ${title}: File doesn't exist`);
              failed++;
              continue;
            }

            const parentItemID = att.parentItemID;
            const newAtt = await Zotero.Attachments.importFromFile({
              file: path,
              parentItemID: parentItemID
            });

            newAtt.setField('title', title);
            await newAtt.saveTx();
            await att.eraseTx();
            await Zotero.Fulltext.indexItems([newAtt.id]);

            results.push(`‚úÖ ${title}: Converted`);
            converted++;
          } catch (e) {
            results.push(`‚ùå ${title}: ${e.message}`);
            failed++;
          }
        } else {
          results.push(`‚è≠Ô∏è  ${title}: Already stored`);
          skipped++;
        }
      }
    }
  }

  results.push(`\n‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê‚ïê`);
  results.push(`‚úÖ Converted: ${converted}`);
  results.push(`‚è≠Ô∏è  Skipped: ${skipped}`);
  results.push(`‚ùå Failed: ${failed}`);

  return results.join('\n');
}

return await convertLinkedToStored();
```

## Innovation Framework

Every summary includes **Innovation Opportunities** using six systematic patterns for extending research:

### 1. Xd - Add a Dimension
Extend the work by adding a new dimension or aspect.

**Examples:**
- Add temporal dimension: Extend 2D image recognition to video/3D
- Add multimodal dimension: Combine vision with audio or text
- Add scale dimension: Make it work at micro or macro levels

### 2. X + Y - Combine
Merge this approach with another technology or method.

**Examples:**
- Combine reinforcement learning + meta-learning
- Combine this model + quantum computing
- Combine approach + blockchain for distributed training

### 3. X|^ - Given a Hammer, Find Nails
Apply this solution to new problem domains.

**Examples:**
- Transformers: Originally for translation ‚Üí Applied to vision, audio, proteins, chemistry
- Find other sequence-based problems: DNA, music, time-series
- Look for domains with similar structure

### 4. X|v - Given a Nail, Find Hammers
Find alternative solutions for this problem.

**Examples:**
- Object detection alternatives: Graph neural networks, capsule networks, neuromorphic
- Explore completely different paradigms
- Question assumptions about the current approach

### 5. X++ - Improve/Increment
Incremental improvements to the current approach.

**Examples:**
- Make it faster: optimization, pruning, distillation
- Make it more accurate: bigger models, better data, ensemble
- Make it more efficient: compression, quantization
- Make it more robust: adversarial training, data augmentation

### 6. Opposite of X - Invert
Explore the inverse or opposite approach.

**Examples:**
- Supervised learning ‚Üí Unsupervised or self-supervised
- Recognition ‚Üí Generation
- Encode-decode ‚Üí Decode-only

**Why This Framework?**
1. **Systematic:** Covers major innovation patterns
2. **Actionable:** Each pattern suggests concrete directions
3. **Creative:** Pushes beyond incremental thinking
4. **Proven:** Framework patterns led to real innovations (ViT, Transformer-XL, etc.)

## FAQ

**Q: Does it work offline?**
A: Yes! Ollama runs locally (fully offline). You only need internet to initially download models.

**Q: Which model should I use?**
A: Start with `llama2` for good balance. Try `mistral` for better quality (slower) or `llama2:7b` for speed.

**Q: Can I process PDFs not in Zotero?**
A: Not currently with the plugin. Use the CLI tool or import PDFs into Zotero first.

**Q: Does it work with group libraries?**
A: Yes! The plugin works with any library you can access in Zotero.

**Q: How do I customize the summary format?**
A: Edit `addon/chrome/content/summarycreator.js` ‚Üí `buildPrompt()` method, then rebuild.

**Q: Why are some sections missing from summaries?**
A: The LLM may not follow the format strictly. Try a larger model like `mistral` for better instruction following.
